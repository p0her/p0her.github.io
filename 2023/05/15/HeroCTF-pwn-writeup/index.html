Finally I solved 3 pwn. (Appointment Book, Impossible v2, Rope Dancer) Personally I feel that Rope Dancer is really fun challenge!

Appointment Book

Analysis

It have 2 function (create_appointment, list_appointment). you can add any timestamp value (integer type) in bss segment, and see the value.

This function name is create_appointment. it do malloc two times of 0x20 size, 0x40 size and input index number. But it occurs out-of-bound vulnerability because don’t check about index. anyway, and input value of date-type (like 2022-12-23 11:12:23) and convert it to timestamp value(integer). after that, it input any string of maximum 62 size in v4.

this function name is list_appointment. this is simple list program. not have any vulnerability.

Vulnerability

As I said earlier it occurs out-of-bound write vulnerability in create_appointment function.

Exploit

So how to use this vulnerability to exploit? It have NX, Stack Canary mitigation so can got overwrite.
and debug_remote function is execute shell! so overwrite any got to debug_remote function.

but we can only input string value, not int value! so we convert debug_remote function address value to date. It is 1970-02-18 14:27:02 (0x401136), and position of appointment’s -5 index is exit@got. so we will use this.

simple exploit xD

Impossible v2

Analysis

we can see format string bug immediately. (sprintf)

The concept of program is AES-encrypt. we should make AES_ECB_128(our_input, key) = 0xdeadbeefdeadbeefcafebabecafebabe.

Vulnerabiltiy

it can format string bug only two times.

Exploit

The key is random via /dev/urandom. but we can overwrite key using format string bug, so we overwrite key to 0x0 and send AES_DECRYPT(0xdeadbeefdeadbeefcafebabecafebabe, 0x0). then can make message is 0xdeadbeefdeadbeefcafebabecafebabe.

Rope Dancer

Analysis

This program is only written in assembly language. (using syscall)

simple sys_write and sys_read, and check that v4 is ‘yes\n’. if correct, call get_motivation_letter.

it occurs stack buffer overflow in sys_read(0, v3, 0x64). If the string you entered is in the form of an email, you can write any string in motivation_letter (bss segment).

Vulnerabiltiy

We can check stack buffer overflow easily.

Exploit

The program is simple but exploit is complicated. First i think about ROP using syscall gadget. yes! it’s sigreturn-rop.
However, since the length of the string that can be entered in the stack is small. So I used stack pivoting to turn rsp to the motivation_letter address, and there I wrote sigreturn rop payload, and we can make ‘/bin/sh’ string in bss easily.
but we have to make eax register to 0xf for call sigreturn syscall. so i spent time that found gadget for this. after few times i found this 2 gadget.

we can make eax register to 0xf using this gadget easily. now we need to stack pivoting gadget. But this program don’t have leave;ret gadget because written by assembly. so i use this gadget for stack pivoting

Then the preparations for the sigreturn rop are done!

  1. set the rbp to motivation_letter - 8
  2. overwrite ret to stack pivoting gadget.
  3. write sigreturn rop payload in motivation_letter.
  4. execute pivoting gadget, then rsp changes to motivation_letter.
  5. execute ‘xor eax, eax; inc al; ret’ and ‘inc al; ret’ 0xe times. then eax register is set to 0xf (sigreturn syscall).
  6. execute sigreturn rop
  7. get shell!
