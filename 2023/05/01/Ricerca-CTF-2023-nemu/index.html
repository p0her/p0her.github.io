pwn

비록 대회할 때 풀지는 못 한 문제지만, 대회가 끝난 후 write-up을 보며 풀었는데 꽤나 재밌었던 문제라 정리를 하려고 한다.

Analysis

문제에서 주어진 코드는 다음과 같다.

평범한 C 프로그램과 다르게 함수안에 함수를 선언하고 사용하고 있다. 그러면 이 때 함수가 어떻게 호출이 되는지 궁금해서 디버깅을 해보았다.

먼저, 각 함수를 선언하면 이것이 심볼로 등록되어, 프로그램에 code 영역에 들어가게 된다.

add 함수를 등록하는 부분 중 일부분을 가져왔는데, 맨 처음에 빈 스택 공간을 rax 레지스터에 등록한 후, rax 에 일정한 오프셋을 더하고, 그 rax 레지스터에 opcode 를 넣게 된다.

opcode 를 살펴보면, r11 , r10 레지스터에 각 각 함수의 주소와, 함수에서 사용하는 변수 (addi 변수에선 a를 사용한다.)의 주소를 넣어주고, 그 함수 주소로 jump를 하는 것을 볼 수 있다. 즉 스택에 opcode가 들어가게 되는데, 발생하는 취약점은 다음 코드에서 볼수 있다.

코드의 일 부분을 가져왔는데, 변수 a 는 int32_t 타입이지만, 전달되는 인수는 uint64_t 타입으로, 4byte overflow가 발생한다. 이걸 어떻게 이용하면 좋을까 디버깅을 해본 결과, 다음과 같은 사실을 알 수 있다.

그러므로, r1 을 4byte overflow하면, add의 opcode 를 조작할 수 있다. 보호기법을 살펴보면 NX도 없으므로, 옵코드를 넣어주면 된다.

exploit

이제 이 옵코드를 어떻게 써먹으면 좋을지 생각해보자. r1, r2, r3, r0 이 연속적으로 배치되어 있으므로, 이곳에 쉘코드를 넣고 점프를 뛰면 된다. write-up을 보면 r0 로부터 차례대로 쉘코드를 넣고, r0 로 jmp를 뛴다. 쉘코드를 살펴보자.

다음과 같이 되어있는데, 디버깅을 해보자

모든 함수가 호출 될 때 다음과 같은 루틴을 거치는데, 우리가 add 함수를 선택했다고 하면, rbp - 0x120 에서 add 함수의 옵코드가 담겨있는 스택주소를 rbx 에 넣어서 call 해주고, rax 에서 rdi 로 값을 옮겨 인자로 사용해준다. 여기서 우리가 r0 을 선택해주면, rax 에 r0 의 주소가 담기고, rbx 는 우리가 아까 r1 을 4byte overflow 함으로써 덮인 옵코드가 실행된다. 이 떄 우리가 jmp rdi opcode를 덮어주면, r0 로 jmp를 뛰게 되는데, r0 에 우리가 아까 언급했던 쉘코드를 차곡차곡 넣어주면, 결국 그 쉘코드가 실행되게 된다. 이제 쉘코드를 설명해보자면,

먼저 rdi 에 0x10을 더해준다. 그 후 rdi 와 rsi 를 바꾸게 되는데, 그러면 rsi는 &r0 + 0x10 의 주소를 가진다. xchg eax, edx 는 eax 에 0을 넣어 read syscall을 호출하게끔 해주려고 하는 것이다. 그 후 syscall 을 하게되면 read 가 실행되게 되는데, 그럼 &r0 + 0x10 에 쉘코드를 넣을 수 있고, 그 후 jmp rsi 를 해주면, rsi 는 &r0 + 0x10 로 뛰게 되고, 쉘코드를 실행하게 된다.

기존에 존재하던 write-up을 거의 베꼈다. 하지만 쉘코드만 실행시키게 하면 OK 이기 때문에, 쉘코드를 작성하는 부분은 맘대로 해주면 될 것이다
